🔄 Complete Request Flow — POST /api/v1/extract
1
index.ts Request enters Express
CORS headers applied. JSON body parsed (up to 10 MB limit). Timestamped request log emitted. Rate limiter checks 60 req/min per req.user.userId — falls back to IP for unauthenticated requests.
2
middleware/auth.ts JWT validation via requireAuth
Reads Authorization: Bearer <token> header. Calls jwt.verify(token, JWT_SECRET) — throws on invalid/expired. On success attaches { userId, email } to req.user and calls next(). Returns 401 on failure.
3
utils/urlValidator.ts URL normalization & platform check
If body doesn't start with "http", extractUrlFromText() grabs first URL via regex (handles TikTok share copy-text like "Check this out! https://vm.tiktok.com/…"). Then isSupportedUrl() tests 4 regex patterns — returns 422 for unsupported domains.
4
services/videoExtractor.ts extractVideoMetadata(url) — yt-dlp subprocess
Spawns yt-dlp with --dump-json --no-playlist --no-warnings. No video downloaded — only JSON metadata. Parses extractor_key to detect platform. Picks best thumbnail URL ≤ 1280px wide. Returns VideoMetadata: title, description, thumbnail_url, uploader, duration, platform, webpage_url.
5
services/imageService.ts downloadAndEncodeImage(url) — thumbnail prep
Axios GET with 10 s timeout + browser User-Agent. Buffer passed to sharp: resize width ≤ 1024 px, JPEG quality 85, withoutEnlargement:true. Returns data:image/jpeg;base64,… string. On any error returns null — thumbnail is always optional.
6
services/claudeService.ts Haiku pre-filter — isRecipeVideo()
Sends title + first 500 chars of caption to claude-haiku-4-5 with max_tokens: 5. System prompt: "Reply with only YES or NO." Answer parsed: if not "YES…" → throws NotARecipeError → route returns 422. This cheap call saves the expensive Sonnet invocation for non-recipe videos.
7
utils/promptBuilder.ts + claudeService.ts Sonnet full extraction
buildExtractionPrompt() assembles a multimodal message: platform, title, creator, duration, caption (up to 3000 chars), plus the base64 thumbnail image block if available. claude-sonnet-4-6 (max_tokens: 4096) responds with a ```json…``` block. parseClaudeResponse() regex-extracts and JSON.parses the block.
8
services/imageGenerationService.ts Miyazaki illustration via Gemini
generateRecipeImage(recipeTitle) calls gemini-2.5-flash-image with a styled prompt: "A beautifully illustrated dish of "[title]". Studio Ghibli / Miyazaki style…". Iterates response parts looking for inlineData. Returns data:image/png;base64,… or null on any error.
9
services/recipeService.ts saveRecipeForUser() — non-blocking DB write
Called with .catch(err => console.error(...)) — does not await. The response is sent immediately; the DB write happens in the background. Prisma nested create: one Recipe record + N Ingredient rows + M Step rows in a single transaction. Uses cuid() for all IDs.
10
routes/recipes.ts JSON response dispatched to iOS client
Full RecipeResponse returned: title, ingredients[], steps[], tags[], extraction_confidence, platform, source_url, thumbnail_base64 (prefers Miyazaki art over original thumbnail). iOS decodes with JSONDecoder().keyDecodingStrategy = .convertFromSnakeCase and stores in SwiftData.

≥ 22.x
yt-dlp
brew install yt-dlp

Local install or Railway URL

For iOS target only
backend/
# 1. Install dependencies
$ cd backend && npm install

# 2. Create .env from example, then fill in your keys
$ cp .env.example .env
# Required: ANTHROPIC_API_KEY, DATABASE_URL, GOOGLE_CLIENT_ID, JWT_SECRET
# Optional: GEMINI_API_KEY (images), INSTAGRAM_COOKIES_B64 (Instagram auth)

# 3. Run Prisma migrations (creates tables)
$ npx prisma migrate deploy

# 4. Start dev server with hot-reload
$ npm run dev
RecipeWizard backend running on http://localhost:8000
  POST http://localhost:8000/api/v1/extract
  GET  http://localhost:8000/health

# 5. Quick health check
$ curl http://localhost:8000/health
{"status":"ok","timestamp":"...","instagram_cookies":"not_configured","database":"ok"}

# 6. End-to-end extract test (need a valid JWT — run auth flow first)
$ curl -X POST http://localhost:8000/api/v1/extract \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-jwt>" \
  -d '{"url":"https://vm.tiktok.com/XXXXXXX","include_thumbnail":false}'

Common Startup Errors

⚠️ ANTHROPIC_API_KEY is not set — recipe extraction will fail
Printed at startup if the env var is missing. The server starts but /extract will throw.
Fix: add ANTHROPIC_API_KEY=sk-ant-... to backend/.env
yt-dlp extraction failed: yt-dlp: command not found
yt-dlp is not installed on the system PATH.
Fix: brew install yt-dlp then verify with yt-dlp --version
P1001: Can't reach database server at localhost:5432
PostgreSQL not running locally or DATABASE_URL is wrong.
Fix: start Postgres with brew services start postgresql or verify Railway DATABASE_URL in .env
npx prisma can pull wrong version (v7 vs v5)
In Dockerfile, npx prisma on npm 11 may download latest Prisma from registry instead of pinned v5.
CLAUDE.md rule: always use node_modules/.bin/prisma directly inside Docker CMD.
