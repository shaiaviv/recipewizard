backend/src/index.ts
Express app entry point — wires up middleware, rate limiting, and routes, then starts the HTTP server.
module level (imports + server bootstrap)
What this file does
Creates the Express application, attaches global middleware (CORS, JSON parsing, logging, rate limiting), mounts all route groups, and starts listening on a port. This is the single entry point — Node runs this file directly with ts-node-dev in dev or node dist/index.js in production.
1import "dotenv/config";Loads .env file into process.env at module load time. Must be the VERY FIRST import so all subsequent code can read env vars.
2import express from "express";Express framework — HTTP server + routing.
3import cors from "cors";Adds CORS headers so iOS app (different origin) can call this API.
4import rateLimit, { ipKeyGenerator } from "express-rate-limit";Rate limiter + IP key generator. Both imported — ipKeyGenerator used as fallback when user isn't authenticated.
5–8import healthRouter from "./routes/health"; import recipesRouter from "./routes/recipes"; import authRouter from "./routes/auth"; import userRecipesRouter from "./routes/userRecipes";Each route module is a separate Express Router. Separation of concerns — each file owns one resource.
10export const app = express();Named export — allows tests (supertest) to import the app without starting the server.
11const PORT = parseInt(process.env.PORT ?? "8000", 10);parseInt with radix 10 prevents octal parsing. Railway injects PORT automatically; defaults to 8000 locally.
14app.use(cors());Allows all origins (default). Sufficient for mobile app — no browser Same-Origin policy applies.
15app.use(express.json({ limit: "10mb" }));10 MB limit because requests may include base64-encoded thumbnail images (~750 KB). Default is 100 KB which would reject them.
18–21app.use((req, _res, next) => { console.log(`[${new Date().toISOString()}] ${req.method} ${req.path}`); next(); });Request logger — prints timestamp, HTTP method, and path to stdout. In Railway, stdout is captured as structured logs.
24–34app.use("/api/v1", rateLimit({ windowMs: 60 * 1000, max: 60, keyGenerator: (req) => req.user?.userId ?? ipKeyGenerator(req.ip ?? "anonymous"), standardHeaders: true, legacyHeaders: false, message: { error: "Too many requests..." }, }));60 requests per minute per USER (not per IP). Rate-limits by userId if authenticated — prevents one user from hammering the API. Falls back to IP for anonymous requests. standardHeaders:true sends RateLimit-* headers (RFC 6585).
37app.use("/", healthRouter);Health check at GET /health — no auth required, used by Railway health probes.
38app.use("/api/v1/auth", authRouter);Public auth routes — no requireAuth middleware. POST /api/v1/auth/google exchanges Google tokens for JWTs.
39app.use("/api/v1", recipesRouter);Mounts /api/v1/extract. requireAuth is applied inside recipesRouter per-route, not globally.
40app.use("/api/v1/recipes", userRecipesRouter);CRUD for saved recipes — GET /api/v1/recipes, DELETE /api/v1/recipes/:id. requireAuth applied inside via router.use().
43–58if (require.main === module) { app.listen(PORT, () => { ... }); }require.main === module is true only when Node runs this file directly (not when imported by tests). Prevents the server from auto-starting during test runs (supertest handles its own binding).
⚠️ Why req.user?.userId ?? ipKeyGenerator(req.ip) — not just IP?
IP-based rate limiting is easily bypassed by users sharing NAT (office, mobile carrier) or using proxies. By keying on the authenticated userId, every user gets their own 60-req/min budget. The ?? nullish-coalescing fallback covers the /auth/google route which runs before requireAuth.

backend/src/middleware/auth.ts
JWT authentication middleware — validates Bearer tokens and attaches the decoded user payload to req.user.
requireAuth(req, res, next)
export function requireAuth( req: Request, res: Response, next: NextFunction ): void
What this function does
Express middleware that acts as a guard. It reads the Authorization header, verifies the JWT using the server's secret, and either stops the request with a 401 error or attaches the decoded payload to req.user and passes control forward. Called by route handlers via router.post("/extract", requireAuth, async ...).
4–7export interface AuthPayload { userId: string; email: string; }The shape of the decoded JWT payload. Defined here so both the middleware and userService.signJwt() agree on the structure.
9–15declare global { namespace Express { interface Request { user?: AuthPayload; } } }TypeScript module augmentation — extends Express's built-in Request type to include the optional user field. Without this, req.user would be a type error. The ? means it's optional — unauthenticated routes won't have it.
17export function requireAuth(req, res, next): void {Return type void is required for Express middleware — TypeScript enforces that you call next() or send a response, not both.
18–21const header = req.headers.authorization; if (!header?.startsWith("Bearer ")) { res.status(401).json({ error: "..." }); return;Optional chaining (?.) handles the case where the header is absent. "Bearer " (with space) is the standard RFC 6750 format. Returns early — MUST return after res.send() to prevent "headers already sent" errors.
22const token = header.slice(7);slice(7) strips "Bearer " (7 characters: B-e-a-r-e-r-space) to get just the token string.
23–28try { const payload = jwt.verify(token, process.env.JWT_SECRET!) as AuthPayload; req.user = payload; next(); } catch { res.status(401).json({ error: "Invalid or expired token" }); }jwt.verify() throws if: signature invalid, token expired, or secret wrong. The ! non-null assertion on JWT_SECRET trusts that startup checked for it (it logs a warning if missing). Cast to AuthPayload is safe because signJwt() always writes that exact shape.
Concept: Module Augmentation (declare global)
TypeScript's declare global lets you add properties to types defined in third-party libraries. Here it extends Express's Request interface so TypeScript knows req.user exists. Without it, req.user.userId would be a compile error — you'd have to use (req as any).user. This pattern is common for attaching custom data to framework request objects.

backend/src/utils/urlValidator.ts
URL pattern matching — validates TikTok/Instagram URLs and extracts URLs from share copy-text.
constants + isSupportedUrl() + detectPlatform() + extractUrlFromText()
export function isSupportedUrl(url: string): boolean export function detectPlatform(url: string): "tiktok" | "instagram" | "unknown" export function extractUrlFromText(text: string): string | null
What this module does
Three URL utilities. isSupportedUrl() validates that a URL is from TikTok or Instagram before spending API credits. extractUrlFromText() handles TikTok's share behavior where the URL is embedded in surrounding copy-text. detectPlatform() returns a string label used for the recipe's platform field.
1–5const TIKTOK_PATTERNS = [ /https?:\/\/(www\.)?tiktok\.com\/@[\w.]+\/video\/\d+/, /https?:\/\/vm\.tiktok\.com\/\w+/, /https?:\/\/vt\.tiktok\.com\/\w+/, ];Three TikTok URL formats: (1) full desktop URL with @username/video/id, (2) mobile shortlink vm.tiktok.com, (3) vt.tiktok.com shortlink used in some regions. All three covered.
7–9const INSTAGRAM_PATTERNS = [ /https?:\/\/(www\.)?instagram\.com\/(reel|p|tv)\/[\w-]+/, ];One regex covers /reel/, /p/ (photo post that can be a video), and /tv/ (IGTV). [\w-]+ matches the base62 post ID.
11–15export function isSupportedUrl(url): boolean { return ( TIKTOK_PATTERNS.some((p) => p.test(url)) || INSTAGRAM_PATTERNS.some((p) => p.test(url)) ); }.some() short-circuits — stops testing after first match. Returns boolean, not the matched platform. Checks both arrays before returning false.
25–28export function extractUrlFromText(text): string | null { const match = text.match(/https?:\/\/[^\s]+/); return match ? match[0] : null; }Regex: https:// or http:// followed by any non-whitespace chars. Handles TikTok iOS share text: "Check out @chef's video! https://vm.tiktok.com/XYZ". match() returns null if no match — ternary returns null.
⚠️ Regex gotcha: /https?:\/\/[^\s]+/ may include trailing punctuation
If a user sends "https://vm.tiktok.com/ABC!" the regex captures the exclamation mark. This is intentional — yt-dlp handles extra trailing chars gracefully. If needed, a stricter regex like /https?:\/\/[^\s.,!?]+/ would trim punctuation. Currently not needed in practice.

The actual recipe is in the caption text, not the video frames.

backend/src/services/imageService.ts
Downloads a thumbnail URL, resizes it to ≤1024px, and returns a base64-encoded JPEG for Claude's vision API.
downloadAndEncodeImage(url): Promise<string|null>
export async function downloadAndEncodeImage( url: string ): Promise<string | null>
What this function does
Downloads the thumbnail from the CDN URL returned by yt-dlp, resizes it using sharp (fast C++ image library), and encodes the result as a base64 data URL string. Returns null on any error — thumbnail is optional, Claude can still extract a recipe from text alone.
4const MAX_WIDTH = parseInt(process.env.THUMBNAIL_MAX_WIDTH ?? "1024", 10);Configurable via env var. Default 1024px. Larger = more detail for Claude but larger base64 payload and higher API cost.
9–14const response = await axios.get(url, { responseType: "arraybuffer", timeout: 10_000, headers: { "User-Agent": "Mozilla/5.0 ..." }, });responseType:"arraybuffer" returns raw binary data instead of a string. 10 second timeout prevents hanging on slow CDNs. Browser User-Agent is required — some CDNs block requests without it.
16–20const buffer = Buffer.from(response.data); const resized = await sharp(buffer) .resize({ width: MAX_WIDTH, withoutEnlargement: true }) .jpeg({ quality: 85 }) .toBuffer();sharp is a C++ binding (libvips) — orders of magnitude faster than pure-JS image processing. withoutEnlargement:true prevents upscaling small images. .jpeg() standardizes format — input might be WebP or PNG. quality:85 good balance of quality vs. size.
22return `data:image/jpeg;base64,${resized.toString("base64")}`;Data URL format: "data:[MIME type];base64,[encoded data]". Anthropic's API expects exactly this format for inline base64 images.
23–26} catch (err) { console.warn("[imageService] Failed...", err); return null; }Swallows any error and returns null. This is intentional — if the thumbnail fails, the recipe extraction continues without an image. Claude's text-only mode is still very accurate.
Concept: sharp's method chain (builder pattern)
sharp(buffer).resize(...).jpeg(...).toBuffer() is a builder pattern. Each method returns the same sharp instance, allowing chaining. The chain is lazy — no processing happens until .toBuffer(), .toFile(), or a pipe is triggered. This is more efficient than running multiple passes over the image data.

parseClaudeResponse(text): RawExtraction
export function parseClaudeResponse( text: string ): RawExtraction
What this function does
Extracts JSON from Claude's response text. Claude is instructed to wrap the JSON in a code fence (```json...```) but sometimes responds with raw JSON if the fence fails. This function tries the regex first, falls back to direct JSON.parse, and throws if both fail.
27const match = text.match(/```json\s*([\s\S]*?)```/);Regex breakdown: ``` literal backticks, json literal, \s* any whitespace including newlines, ([\s\S]*?) capture group matching ANY character (including newlines) lazily, ``` closing backticks. Important: Two-stage AI cost optimization pattern
Stage 1 (Haiku, ~$0.0003): "Is this a recipe? YES/NO" — runs on title + 500 chars of caption only. Stage 2 (Sonnet, ~$0.003): Full multimodal extraction — runs only if Stage 1 says YES. For a typical app with 50% non-recipe shares, this halves the AI cost without reducing output quality. The pattern generalizes: use a cheap classifier to gate an expensive generator.

Pattern: Multimodal message construction (image + text interleaving)
Anthropic's messages API supports interleaved content blocks: [{type:"text",...}, {type:"image",...}, {type:"text",...}]. Text before the image provides context; the image is the visual input; text after guides interpretation. This ordering mirrors how humans read — "here's the context, here's the visual, here's what I need from it."

export async function findOrCreateUser( info: GoogleUserInfo ): Promise<User>
What this function does
Creates a new user or updates an existing one in PostgreSQL, keyed by googleId. The "upsert" operation is atomic — no race condition between "does user exist?" and "create user". This is the idempotent user registration pattern.
